The bits and bolts

For most, if not all, the functionality javascript was used. The main problem which was evaluating tournaments is solved using a recursive function that evaluates sub-tournaments until a single match is found. After that, it connects to the Parse database to update the scores of the players. It is all serverd from a AWS EC2 free server that lets you host a small website.

Why so technological?

Javascript and frameworks: Most of the solution was coded in plain js to keep things simple. The Parse Javascript SDK was used to connect to the database, because of its power and simplicity.

Design: the tecnologies used here were CSS3 and HTML mostly, because what we needed was a website. Other tools were used in the design process like: GIMP 2 (asset creation), Codepen (test the website on the fly), Sumblime Text 2(editor)

Last but not least I used Amazon EC2 server because they are free for a whole year and easy to set up (I had done it before). To host the code I used GitHub as sugested and because of the hype thats building around it.
